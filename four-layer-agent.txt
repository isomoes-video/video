中文标题：多 Agent 四层工作流拆解
English Title: Breaking Down a Four-Layer Multi-Agent Workflow

这期视频围绕作者团队最近重组的一套 multi-agent 工作流展开，核心出发点是成本终于降下来了。过去多 agent 并行会带来远超线性增长的 token 开销，原本每月几十美元的使用成本很容易膨胀到数百美元，因此很多想法一直停留在尝试阶段。随着 GPT Codex Team Plan 一类更低成本方案出现，团队重新把多 agent 编排、并行运行和人工审核整合起来，形成了一个更可落地的实践框架。

视频将整个系统拆成四层来讲：最底层是更便宜的模型与账号池，解决预算问题；第二层是作为 agent runtime 的 OpenCode，负责 session、工具调用和权限控制；第三层是 SuperPower 这一层的多 agent 能力，包括 git worktree 隔离、subagent 并行、自动生成 plan 并按步骤推进任务；最上层则是团队自建的 AI 看板，用于查看历史记录、审查执行过程、验收结果并做关键决策。视频也强调，并不是所有任务都值得引入 SDD/TDD 或复杂编排，简单任务过度工程化反而会让结果变差；真正适合这套体系的，是上下文长、步骤多、需要长期运行和并行协作的复杂任务。整体来看，这套四层结构的价值，不在于完全替代人，而在于把人从具体编码中抽离出来，更多承担规划、审阅、取舍和质量控制的角色，从而显著提升多任务并行处理效率。
