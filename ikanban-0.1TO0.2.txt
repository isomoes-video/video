中文标题：爱看版 0.1.6→0.2.7 升级解读
English Title: iKanban 0.1.6 to 0.2.7 Upgrade Overview

这期视频主要介绍ikanban从 0.1.6 升级到 0.2.7 的核心变化。作者首先说明此次升级的主线是架构调整：从“Web + 自定义后端 runtime”的形态，切换为“纯前端 Web 客户端 + OpenCode Server”模式。新版直接部署在 GitHub Pages，用户可通过静态链接访问，不再要求本地安装前端服务，整体使用门槛明显降低。


在功能层面，视频重点展示了多项目并行工作、聊天与 Review 一体化界面、以及 keyboard-first 的交互方式（快捷键与命令面板）。在技术层面，作者解释了重构后的取舍：前后端解耦更彻底、部署更轻量、可更方便适配多端（如 Windows、iOS、移动端），但也失去了一部分依赖本地后端/本地 Git 引擎的能力，当前 Diff 更多依赖 OpenCode API 返回。对于后续路线，考虑通过服务器能力、新 API 或补充后端来弥补能力缺口，并继续推进多引擎调度等扩展。结尾回顾了版本升级价值，并呼吁用户体验后支持项目。
