中文标题：isomoes api 接入 Codex 配置指南
English Title: How to Connect isomoes api to Codex

这期视频围绕如何将 isomoes api 接入 Codex 展开，重点讲解 Codex CLI 与桌面版共用同一套配置的方法。视频先说明 Codex 的 CLI 与桌面端在配置逻辑上是一致的，因此只需要完成一次持久化配置，就可以同时在终端和桌面环境中使用。安装部分没有展开细讲，只简单提到如果本机已经具备 Node.js 运行环境，那么通过 npm 安装 Codex 会比较直接。

随后，视频把重点放在 `~/.codex/config.toml` 这一核心配置文件上，演示如何将示例内容复制进去，并提醒几个容易出错的细节：接口 URL 需要替换为 isomoes api 提供的地址，模型提供商名称要与模型配置保持一致，否则无法正常识别；同时 Codex 不能像某些其他工具那样直接把密钥写入配置文件，而是必须通过环境变量读取 API Key。

在环境变量部分，视频分别说明了不同操作系统下的做法。macOS 和 Linux 用户可以通过 `export` 的方式将密钥写入 shell 环境，并持久化到 `bashrc` 或 `zshrc`；Windows 用户则需要在系统环境变量中新增对应的 Key。完成这些准备后，视频通过实际运行 Codex 进行检测，验证 CLI 可以正常调用模型，再进一步验证桌面端也能正确读取同一套配置并完成检测。最后总结强调，只要配置文件和环境变量都设置正确，一套配置即可同时支持 Codex CLI 与 Codex Desktop。
