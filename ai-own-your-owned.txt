中文标题：让 AI Agent 拥有人的验证能力
English Title: Let AI Agents Own Human Validation Capabilities

这期视频围绕一个核心观点展开：如果 AI Agent 想真正承担更长、更复杂的任务，它就必须拥有一部分原本只属于人的能力，尤其是观察、验证、调试和反馈的能力。视频先回顾了近几年 AI 编程工具的发展路径，从问答式辅助、自动补全，到编辑器内对话，再到能够读写文件、执行脚本和完成测试的 agent 模式。随着上下文越来越充分、工具越来越完善，单个 agent 在明确意图和高质量上下文的支持下，已经可以独立完成不少任务。但问题在于，这种能力仍高度依赖 human-in-the-loop，也就是人在每一轮中不断授权、判断和纠偏。

视频进一步指出，这种工作方式在单任务场景下尚可接受，但一旦进入 multi-agent 系统，人就会迅速成为系统带宽的瓶颈。因为多个 agent 并行运行时，如果每一个小任务、每一轮循环都需要人工 review 和反馈，那么人的审查能力就限制了整个系统吞吐量。基于这个判断，作者提出应尽量把人从局部的小循环中移出，不是完全取消人的作用，而是把人工参与从“每一步都介入”改为“只在关键节点介入”。这意味着 agent 需要具备自我验证能力，能够在一定边界内自己发现问题、修正问题并持续迭代，直到结果达到要求，或者超出自身能力范围后再交给人类接手。

为了说明这一点，视频用网页开发场景做了具体示例。作者认为，当 agent 接入浏览器、截图、DOM 检查、HTTP 请求监控等工具后，它就不再只是“会写代码”，而是具备了接近专业用户或开发者的视角。这样一来，agent 在接到需求后，可以先写代码，再自动打开页面、执行操作、观察结果、检查渲染与交互是否正常；如果结果不对，就继续修改代码并重复验证流程。过去这类循环中每一步都需要人介入，如今则可以由 agent 自行完成大部分迭代，只有最终验证通过或长时间无法修复时，才通知人工 review。这种机制本质上是在建立 agent 的 validation loop，让它拥有部分“人原本拥有的东西”。

视频最后总结，当前 AI Agent 的能力虽然仍有明显边界，人类也依旧不可替代，但人的角色正在改变。随着模型更强、成本更低，multi-agent 可能成为下一阶段的重要方向。未来真正有价值的，不只是让 agent 会做任务，而是让多个 agent 在尽量少的人类干预下协同完成更长程、更复杂、甚至非线性的工作。同时，人也需要持续学习新的知识与协作方式，因为一旦某种能力被 agent 吸收，人的价值就必须向更高层次迁移。
