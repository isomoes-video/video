中文标题：爱看板使用指南与功能演示
English Title: iKanban Usage Guide and Feature Demo

这期视频围绕“爱看板”项目新增的看板功能展开，重点演示如何完成基础接入与日常使用流程。视频先说明看板已经可以直接通过对应链接访问，然后介绍使用前最关键的一步：连接后端服务。讲解中提到，通常会在本地启动一个 OpenCode Server，并根据实际环境区分代理与非代理方式，同时配置需要暴露的端口以及 CORS 跨域参数，确保前端面板能够顺利连接到服务端。

在完成服务连接后，视频继续展示如何选择一个实际项目进行操作。讲者通过进入目标项目目录、复制路径并在看板中打开对应 project 的方式，演示了如何把某个具体工程接入到面板里开始使用。随后视频进一步体现“看板”形态的核心价值：不仅可以在一个项目里发起任务，还可以同时打开多个不同 project，并行执行不同检查或查询任务，从而直观看到哪些任务正在处理中、哪些任务已经完成。

后半部分则介绍了任务流转和结果处理逻辑。视频说明当前版本已经支持多项目并行，但同一项目下基于 git worktree 的多分支能力暂未加入，后续可能补充。对于已完成的任务，用户既可以继续对话、追问细节，也可以直接归档，使一次对话流程闭环。整体上，这套看板的核心工作方式就是：发起对话、查看执行结果、进行 review、决定通过或继续追问，最后归档收尾。整期内容偏向功能介绍与上手说明，适合第一次接触爱看板的用户快速理解其使用方式与设计思路。
