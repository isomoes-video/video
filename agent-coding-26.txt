中文标题：AI Coding 演进与多 Agent 未来
English Title: The Evolution of AI Coding and the Multi-Agent Future

这期视频围绕过去一年多 AI Coding 的变化展开回顾，试图回答一个核心问题：从早期的 Web Coding、Copilot 补全，到如今具备自动读写文件、运行命令、连接外部系统的 Agent，软件开发这件事究竟发生了什么本质变化。视频先从创作者自己的使用经验切入，回顾从纯手写代码、局部补全、对话式辅助，到 Agent 自动调用工具的完整演进路径，指出开发过程正在从“人主导、AI 辅助”逐步转向“AI 主执行、人做决策”。

随后，视频将这种变化拆成三个层次。第一层是单个 Coding Agent 的出现，它不再只是一次提问一次回答的聊天工具，而是能够在单次任务中连续思考、调用工具、修改文件并反馈结果。第二层是 Agent 能力扩展的问题，也就是通过 MCP 与 Skills 这类机制，让模型突破自身训练数据的限制，连接文档、浏览器、办公软件乃至真实生产环境，从单一代码助手扩展成更通用的执行系统。视频对这两条技术路径做了对比：MCP 更像标准化、偏安全的外部能力协议，Skills 则更像轻量、易分发、渐进加载的提示词能力包，两者各有优势，也分别对应不同的生态扩张方式。

在第三层，视频把重点放在 Multi-Agent 协作上。作者认为，随着单个 Agent 越来越强，真正拉开人与人效率差距的，不再只是会不会写代码，而是能不能拆解任务、组织流程、调度多个 Agent 并行工作。无论是基于 worktree 的隔离执行，还是借助 AI 看板去分配任务、合并结果，本质上都在探索“一个人如何借助多个 Agent 放大产能”。在这样的开发范式中，人的角色也会随之改变，更偏向架构设计、方案评审和关键决策，而不再停留在大量重复性的手工编码上。

最后，视频将这些观察落到对 2026 年的判断上：AI Coding 仍然只是起点，接下来更重要的方向会是 Agent Teams、工程化工作流、模块化任务组织，以及面向 AI 原生应用的基础设施建设。随着工具、协议、工作流和协作方式继续成熟，未来不仅会出现更多服务于 AI 的软件，也会诞生一批真正由 Agent 驱动、以 AI 为核心生产力的原生应用。
